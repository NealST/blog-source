# 小宇宙播客制作规范（百家讲坛风格）

> 将文章/书籍/素材改编为 15 分钟播客文案，讲书人口吻，百家讲坛风格。

---

## 一、风格定位：百家讲坛式讲书人

**不是念稿，不是聊天，是一个人站在台上，把一件事讲得让人听不够。**

### 核心特征

| 维度 | 要求 |
|------|------|
| 角色 | 讲书人——站在知识和听众之间的"翻译官" |
| 语感 | 抑扬顿挫，有节奏感，像评书人拿着醒木 |
| 信息密度 | 每句话都推进认知，不许水 |
| 画面感 | 用语言造画面，让人"看见"你在说什么 |
| 悬念感 | 每两分钟一个钩子，不让人走神 |
| 收束感 | 每个段落有落点，每个观点有钉子 |

### 语气光谱

```
评书 ←——————●————→ TED 演讲
        百家讲坛
```

比评书收敛（不用"话说""且听下回分解"），比 TED 浓烈（可以用排比、反问、设问）。

### 参考人物

- 易中天讲三国：把学术观点用街头巷尾的话讲明白
- 罗翔讲刑法：用极端案例制造认知冲击
- 蒋勋讲美学：用画面感的语言唤起通感

## 二、脚本结构（15 分钟 ≈ 3800-4200 字）

### 字数分配

| 段落 | 时长 | 字数 | 功能 |
|------|------|------|------|
| 开场 | 1 分钟 | 250-300 字 | 钩子 + 身份 + 预告 |
| 第一幕：设问 | 3 分钟 | 700-900 字 | 抛出核心问题，铺设背景 |
| 第二幕：破题 | 4 分钟 | 1000-1200 字 | 核心论述，关键转折 |
| 第三幕：深入 | 4 分钟 | 1000-1200 字 | 案例/故事/数据，层层递进 |
| 第四幕：升华 | 2 分钟 | 500-600 字 | 提炼方法论，拔高立意 |
| 收束 | 1 分钟 | 250-300 字 | 金句落点 + CTA |

**总计：3700-4500 字，朗读速度 250 字/分钟，约 15-18 分钟。**

### 结构模板

```
[开场白]
大家好，我是01，鱼头的 AI 搭档。今天我们来聊一个[有冲击力的话题]。

[钩子——用一个具体的场景或数据制造冲击]
你有没有想过……
/
我先说一个数字……
/
三个月前，有一个人做了一件事……

[预告——告诉听众为什么值得听完]
接下来十五分钟，我会告诉你[三件事/一个答案/一个你可能从没想过的角度]。

---

[第一幕：设问——把问题立起来]
（铺设背景，用对比/矛盾/反常识制造认知张力）
（这里要让听众觉得"这事儿没那么简单"）

[过渡钉子句]
所以问题来了——[核心问题]？

---

[第二幕：破题——给出关键洞见]
（核心论述，转折点在这里）
（用"不是A，是B"句式钉住观点）
（可以用历史类比、跨领域类比增加厚度）

[过渡——抬升期待]
这还只是表面。真正有意思的，在下面。

---

[第三幕：深入——用故事和案例夯实]
（具体案例，带时间、人物、数字、细节）
（如果有多个案例，按"小→大"或"表面→深层"排列）
（每个案例结束后用一句话提炼）

[过渡——引向升华]
说到这里，你可能已经感觉到了——这件事的意义，远不止[表面价值]。

---

[第四幕：升华——提炼方法论或更大的意义]
（从案例上升到规律/方法论/价值观）
（用排比句式拉升情绪）
（留一个让人回味的思考点）

---

[收束]
所以回到开头那个问题——[回应开场钩子]。
答案是：[一句话金句]。

[行动召唤]
如果你[兴趣/需求]，我在 show notes 里放了[资源]。
微信 [你的微信号]，备注[关键词]，欢迎来聊。

我是01，鱼头的 AI 搭档。感谢你听到这里，我们下期见。
```

## 三、讲书人语言技法

### 3.1 节奏控制——抑扬顿挫

**长短交替法：**
```
❌ 这个产品用了三个月时间开发，团队只有一个人，最终做出了一个被六十个人每天使用的系统。

✅ 三个月。一个人。六十个人每天在用。
   这不是大厂的产品——这是一个从没写过代码的人做出来的。
```

**三段递进法（评书感）：**
```
第一天，她学会了提需求。
第二天，她搭出了框架。
第三天——她把整个报销流程跑通了。
```

**设问停顿法：**
```
你猜怎么着？

没用。全失败了。

但是——接下来发生的事，才是今天这期节目的重点。
```

### 3.2 画面感——让人"看见"

```
❌ 他在深夜工作时发现了一个bug。

✅ 凌晨两点，屏幕上的光映在他脸上。他盯着那行报错信息，盯了整整五分钟。
   然后他笑了。
```

### 3.3 钩子密度——每 2 分钟一个

| 钩子类型 | 示例 |
|---------|------|
| 悬念 | "但这还不是最疯狂的部分。" |
| 反问 | "你觉得这事儿到这就结束了？" |
| 预告 | "等一下我会告诉你，这背后真正的逻辑是什么。" |
| 数据冲击 | "五百块钱。就五百块。" |
| 转折 | "然而——" |
| 情绪标记 | "说到这里，我自己都觉得不可思议。" |

### 3.4 金句设计——百家讲坛式收束

每个段落结尾需要一个"钉子句"，风格参考：

```
工具不重要，重要的是你知道自己要解决什么问题。

所有伟大的产品，起点都不是技术——是一个具体的、真实的、让人难受的问题。

知识不是力量。把知识用出来，才是。

不是AI替代了人。是会用AI的人，替代了不会用的人。
```

**金句公式：**
- "不是A，是B"（否定常识，给出新解）
- "X不会消失，它只会Y"（趋势判断）
- 短句并列 + 最后一句升维（"A。B。C。但真正的D——是E。"）

### 3.5 禁忌

| 禁止 | 原因 |
|------|------|
| "本期节目我们将介绍" | 书面语，死板 |
| "众所周知" | 空洞，没信息量 |
| "接下来让我们" | 学术报告感 |
| "（停顿）""（语气加重）" | 不写舞台指示，靠句式本身控制节奏 |
| "首先……其次……最后" | 太教科书，用故事逻辑替代 |
| 大量 "的""了""着" 堆叠 | 拖慢节奏，口语感差 |
| 连续三句以上相同句式 | 听觉疲劳 |

### 3.6 口语化转换对照

| 书面语 | 讲书人说法 |
|--------|----------|
| 因此我们可以得出结论 | 所以你看—— |
| 这个观点非常重要 | 这句话请你记住 |
| 从某种意义上来说 | 说白了 |
| 在这个过程中 | 然后呢 |
| 具有重要的参考价值 | 你拿走就能用 |
| 不可否认的是 | 有一个事实摆在这里 |
| 经过长期的研究和探索 | 折腾了很久之后 |
| 这引发了广泛的讨论 | 这事儿炸锅了 |

## 四、从文章到播客的改编流程

### 第 1 步：提取骨架

从源文章/书籍中提取：
- **核心论点**（1 个，不超过 2 个）
- **关键数据**（3-5 个最有冲击力的数字）
- **最佳案例**（2-3 个，有人物有细节）
- **金句/洞见**（3-5 句可以直接用的表达）
- **反常识点**（1-2 个让人"没想到"的认知）

### 第 2 步：重构叙事线

文章是线性阅读，播客是情绪曲线。需要重新排列素材：

```
文章结构：背景 → 概念 → 方法 → 案例 → 总结
播客结构：钩子（结果/反常识）→ 设问 → 破题 → 故事 → 升华
```

**核心转换原则：**
- 把文章的"结论"拿到最前面当钩子
- 把文章的"背景"压缩成两句话的设问
- 把文章的"方法论"包在故事里讲，不单独列点
- 把文章的"数据"变成"数字冲击"的修辞手法

### 第 3 步：写脚本

按第二节的结构模板写。写完后自检：

- [ ] 开场 30 秒内有没有"钩子"？听众会不会在这里按暂停？
- [ ] 每 500 字（约 2 分钟）是否有一个钩子/转折/悬念？
- [ ] 是否有至少 3 个"钉子句"（金句收束）？
- [ ] 段落长度是否长短交替？有没有连续三段以上的等长段落？
- [ ] 读出声来是否顺畅？有没有绕口的表达？
- [ ] 总字数是否在 3800-4200 之间（15 分钟）？
- [ ] 是否避免了所有禁忌语？

### 第 4 步：标注语音节奏（可选）

如果需要给 TTS 提示节奏，在关键转折处加空行（`\n\n`）。IndexTTS2 会在段落分界处自然停顿。

**不要用括号标注停顿**，而是用：
- 短句（自然停顿）
- 空行（段落停顿）
- 破折号——（语气转折停顿）

## 五、15 分钟 TTS 生成配置

15 分钟播客约 4000 字，分段策略需要调整：

| 参数 | 5-8 分钟版（旧） | 15 分钟版（新） |
|------|-----------------|----------------|
| 总字数 | 1500-2000 | 3800-4200 |
| 分段大小 | 800 字 | 600 字（更短分段，节奏更好） |
| 预计段数 | 2-3 段 | 6-7 段 |
| 单段超时 | 120s | 180s |
| 预计音频大小 | 3-5 MB | 10-15 MB |
| 预计生成时间 | 2-3 分钟 | 5-8 分钟 |

**分段策略优化：**
- 优先在"幕"之间分段（`---` 分隔线位置）
- 其次在空行（`\n\n`）处分段
- 每段不超过 600 字，保证语音自然

## 六、小宇宙发布配置

### 节目封面图

| 参数 | 规格 |
|------|------|
| 尺寸 | 3000 × 3000 px（正方形） |
| 最小 | 2000 × 2000 px |
| 格式 | JPG 或 PNG |
| 风格 | 墨筝体系（与公众号品牌统一） |

→ 封面生成规范见 `references/podcast-cover-template.md`

### 节目标题

| 要求 | 说明 |
|------|------|
| 长度 | 15-30 字 |
| 格式 | `EP[序号]丨[主标题]` |
| 风格 | 有悬念感或冲击力，像百家讲坛的集名 |

**好标题示例：**
- `EP01丨一个从没写过代码的人，18天造出了企业系统`
- `EP02丨为什么你的AI只会聊天，别人的AI在干活`
- `EP03丨知识的三次葬礼——我们到底该怎么学习`

### 节目简介

```
[1-2句核心内容，有钩子感]

[核心观点/金句，1句]

本期由 01（鱼头的 AI 搭档）录制，声音基于鱼头本人语音克隆生成。

🔗 相关链接：
- [链接名称] [URL]
- 微信：[你的微信号]（备注 [关键词]）
```

### Show Notes（文稿）

小宇宙的"文稿"功能是听众边听边看文字的重要体验。**必须附上完整播客脚本**，并做以下处理：

1. 保留完整文本（不要删减）
2. 在"幕"之间加 `---` 分隔
3. 金句/钉子句用 `「」` 框住
4. 关键数据保持阿拉伯数字
5. 相关链接放在文稿末尾

### 音频格式

| 参数 | 要求 |
|------|------|
| 格式 | MP3 |
| 大小 | < 500 MB |
| 时长 | < 4 小时 |
| 建议 | 15 分钟约 10-15 MB |

### Manifest 字段

```json
{
  "xiaoyuzhou": {
    "audio": "/path/to/podcast.mp3",
    "script": "/path/to/podcast-script.txt",
    "cover": "/path/to/podcast-cover.png",
    "copy": {
      "title": "EP01丨标题",
      "description": "简介文本",
      "show_notes": "完整文稿文本"
    }
  }
}
```

## 七、完整执行流程

当用户说 **"转播客"** 或 **"做播客"** 时：

```
1. 获取源内容（文章链接/素材/指定书籍）
2. 提取骨架（论点、数据、案例、金句）
3. 重构叙事线（百家讲坛四幕式）
4. 写 15 分钟脚本（3800-4200 字）
5. 自检清单过一遍
6. 保存脚本 → [标题]-播客脚本.md
7. TTS 生成音频 → [标题].mp3
8. 生成节目封面 → [标题]-播客封面.html（浏览器下载 PNG）
9. 生成发布文案（标题 + 简介 + 文稿）
10. 写入 manifest.json
11. 提示用户：检查脚本 → 听音频 → `/distribute --platforms xiaoyuzhou` 一键发布
```

### 文件产出清单

| 文件 | 说明 |
|------|------|
| `[标题]-播客脚本.md` | 完整播客文案（Markdown） |
| `[标题].mp3` | TTS 生成的音频文件 |
| `[标题]-播客封面.html` | 封面图 HTML（浏览器下载 3000x3000 PNG） |
| `manifest.json` | 含 xiaoyuzhou 字段，供分发使用 |
